# BM25 Lexical Retrieval Reference

*Library: `rank_bm25` — pure Python, no external index required. Install: `pip install rank-bm25`*

---

## The Non-Negotiable Rule: Symmetric Tokenization

The single most common BM25 bug: **different tokenization for corpus vs query**. BM25 is a
term-matching algorithm. If the corpus is tokenized one way and the query another, terms
that should match don't, and recall silently collapses.

**Rule:** apply *exactly* the same preprocessing pipeline (tokenizer, lowercasing, stop-word
list, stemmer) to every document at index time AND to every query at search time.

---

## Basic Usage

```python
from rank_bm25 import BM25Okapi

corpus = [
    "The quick brown fox jumps over the lazy dog",
    "Never jump over the lazy dog quickly",
    "Brown foxes are quick and agile animals",
]

# Step 1: tokenize corpus — store tokens, not raw text
tokenized_corpus = [doc.lower().split() for doc in corpus]
bm25 = BM25Okapi(tokenized_corpus, k1=1.5, b=0.75)

# Step 2: tokenize query with THE SAME pipeline
query = "quick fox"
tokenized_query = query.lower().split()  # must match corpus tokenization

# Get scores for all documents
scores = bm25.get_scores(tokenized_query)       # np.ndarray shape (len(corpus),)

# Get top-n document strings
top_docs = bm25.get_top_n(tokenized_query, corpus, n=3)
```

## Parameters

### k1 — Term Frequency Saturation (default 1.5)

Controls how quickly additional occurrences of a term stop adding value.

- **Low k1 (0.5–1.0):** term frequency saturates fast; a term appearing 3× isn't much
  better than appearing once. Good for short documents.
- **High k1 (1.5–2.0):** term repetition keeps contributing. Better for long, detailed
  documents where repeated mentions signal importance.
- **Recommended starting point:** k1=1.5. Tune up toward 2.0 if your documents are long
  and term repetition is meaningful.

### b — Length Normalization (default 0.75)

Controls how much document length penalizes term-frequency scores.

- **b=0.0:** no length normalization; long docs always score higher.
- **b=1.0:** full normalization; all docs treated as equal length.
- **b=0.75:** the standard default — partial normalization. Usually leave it here.
- **Lower b (toward 0.5):** better for domains where longer documents are genuinely more
  informative (e.g., research papers).

## Preprocessing Pipeline

Heavier preprocessing improves recall for morphologically rich queries but adds complexity.
Build incrementally: start with lowercase-only, add stop-words if precision is low,
add stemming if recall is low.

### Level 1 — Lowercase + whitespace tokenization (minimum)
```python
def tokenize(text: str) -> list[str]:
    return text.lower().split()
```

### Level 2 — Lowercase + punctuation removal + stop-words
```python
import re

STOP_WORDS = {"the", "a", "an", "is", "in", "on", "at", "to", "for", "of", "and", "or"}

def tokenize(text: str) -> list[str]:
    text = re.sub(r"[^\w\s]", "", text.lower())
    return [w for w in text.split() if w not in STOP_WORDS]
```

### Level 3 — Level 2 + stemming (Porter stemmer)
```python
import re
from nltk.stem import PorterStemmer

stemmer = PorterStemmer()
STOP_WORDS = {"the", "a", "an", "is", "in", "on", "at", "to", "for", "of", "and", "or"}

def tokenize(text: str) -> list[str]:
    text = re.sub(r"[^\w\s]", "", text.lower())
    return [stemmer.stem(w) for w in text.split() if w not in STOP_WORDS]
```

**Critical:** whichever level you choose, use the **exact same `tokenize` function** for
both indexing and querying.

## How BM25 Scoring Works (For Tuning Intuition)

BM25 score for query Q and document D:

```
score(D, Q) = Σ_i  IDF(qi) * (tf(qi,D) * (k1+1)) / (tf(qi,D) + k1*(1-b + b*|D|/avgdl))
```

- `tf(qi, D)` — frequency of query term qi in document D.
- `|D|` — document length in tokens; `avgdl` — average document length in corpus.
- `IDF(qi) = ln((N - n(qi) + 0.5) / (n(qi) + 0.5) + 1)` where N = corpus size,
  n(qi) = number of documents containing qi. Rare terms get high IDF; ubiquitous
  terms approach 0.

## Variants: BM25L and BM25+

`rank_bm25` ships three variants:

```python
from rank_bm25 import BM25Okapi, BM25L, BM25Plus
```

| Variant | Use case |
|---|---|
| `BM25Okapi` | Default — correct for most tasks |
| `BM25L` | Reduces over-penalization of long documents; try when long docs rank too low |
| `BM25Plus` | Prevents score from dropping to 0 for terms with very low TF; can improve recall |

## Full Working Example (with Stop-words)

```python
import re
from rank_bm25 import BM25Okapi

STOP_WORDS = {"the", "a", "an", "is", "in", "on", "at", "to", "for", "of", "and", "or"}

def tokenize(text: str) -> list[str]:
    text = re.sub(r"[^\w\s]", "", text.lower())
    return [w for w in text.split() if w not in STOP_WORDS]

corpus = [
    "Python releases the GIL for IO-bound tasks but not CPU-bound operations.",
    "NumPy bypasses the GIL by calling into C extensions directly.",
    "Using asyncio allows concurrent IO without threads.",
]

tokenized_corpus = [tokenize(doc) for doc in corpus]
bm25 = BM25Okapi(tokenized_corpus, k1=1.5, b=0.75)

query = "GIL and Python threading"
tokenized_query = tokenize(query)  # same tokenize function
scores = bm25.get_scores(tokenized_query)
top_docs = bm25.get_top_n(tokenized_query, corpus, n=2)

for doc in top_docs:
    print(doc)
```
